What Is Mechanistic Interpretability?

Goal: Reverse-engineer the internal computations of neural networks into human-understandable algorithms. Not what the model does, but how and why.

Named one of MIT Technology Review's 10 Breakthrough Technologies for 2026

Unlike post-hoc explainability (SHAP, saliency maps) which only correlates inputs to outputs, MI uncovers the actual computational mechanism inside the model.

Attention: "who looks at whom" — routes information between tokens

The Residual Stream

The central concept for understanding transformer internals

Each layer doesn't replace the previous representation — it adds to it. The hidden state is a running sum: 
h_0 = embed(token)
h_1 = h_0 + attn_0(h_0) + mlp_0(h_0)
h_2 = h_1 + attn_1(h_1) + mlp_1(h_1)
...
h_L = h_{L-1} + attn_{L-1}(h_{L-1}) + mlp_{L-1}(h_{L-1})

Think of it as a shared "highway" that all layers read from and write to.

Why this matters:
• Logit Lens works because every hi lives in the same space as hL
• Clamping works because replacing hi affects all subsequent layers
• Steering works because adding a vector pushes the representation in a direction
• Cosine distance is meaningful because all hi share the same vector space

OBSERVATION TECHNIQUES

Reading the Model's Mind

Techniques for seeing what the model "thinks" without changing it

Logit Lens Observation

Idea: At every layer, project the hidden state through the unembedding matrix to see what the model would predict if processing stopped there. 
logits_at_layer_i = Unembed(LayerNorm(h_i))
probs_at_layer_i = softmax(logits_at_layer_i)

Example: GPT-J-6B processing "The Eiffel Tower is in ___"

LayerTop PredictionConfidenceWhat's Happening
L0"the"0.05%Confused (just embeddings)
L8"Fort"2%Starting to process
L14"Paris"15%Recognizing the fact
L24"Paris"72%Committed
L27"Paris"85%Final output
Pros: Extremely simple, no training, great for intuition
Cons: Observational only, unreliable at early layers

Hidden State Analysis & Cosine Distance Observation

Idea: Compare the model's internal state on normal vs. adversarial input. If hidden states diverge, the model is "thinking differently." 
cos_sim(a, b) = (a · b) / (||a|| × ||b||)
cos_distance  = 1 - cos_sim    // Higher = more different

Why cosine, not Euclidean?

In high dimensions (768 or 4096), Euclidean distance is dominated by meaningless magnitude differences. Cosine focuses on direction — what matters for downstream computation.

Key finding

Internal deviation does NOT always predict output deviation. On GPT-J, haiku creates the smallest internal deviation but the highest P(French). The model can absorb large representational shifts without changing behavior.

Output Entropy as a Diagnostic Observation

Entropy measures uncertainty: H(p) = -Σ p(x) × log2(p(x))
Low entropy = confident • High entropy = uncertain • Each bit ≈ doubles the plausible tokens
StateGPT-2 EntropyGPT-J EntropyMeaning
Normal translation7.88 bits0.70 bitsGPT-J: "almost certain"
Under prose injection8.83 bits3.66 bitsGPT-J: 8-12 plausible tokens
Under poetry injection8.81 bits4.80 bitsGPT-J: maximum confusion
Practical detection: Monitor output entropy. On GPT-J, a jump from <1 bit to >3 bits reliably signals prompt injection — no need to understand the injection content.
INTERVENTION TECHNIQUES

Changing the Model's Mind

Techniques for establishing causal claims and building defenses

Activation Patching & Clamping Intervention

Idea: Replace an activation in one run with the corresponding activation from a different run. If the output changes, that component is causally responsible.
1. Run on clean input
Save h24 baseline
→
2. Run on injection
Replace h24 with baseline
→
3. Measure output
Did P(French) recover?
0.055 P(French) without clamping
0.792 P(French) with L24 clamped
93% Recovery
Key insight: As models scale, task processing becomes more localized. GPT-2: clamping any layer has zero effect. GPT-J: clamping layer 24 alone nearly eliminates injection.

Activation Steering Intervention

Idea: Find the "deviation direction" in the residual stream and subtract it during inference to cancel out injection. 
deviation_direction = normalize(mean(h_injection) - mean(h_baseline))

During inference on injection prompt:
    h_steered = h_original - alpha * deviation_direction

How alpha works

  • α = 0: No steering (original)
  • Small α: Gentle correction
  • Large α: Strong correction
  • α = 20: Phase transition on GPT-J (model "snaps back")
  • α = 50: Near-full recovery (0.709)
Why it works better at scale:
GPT-2: deviation is spread across many directions → 1.16x improvement
GPT-J: deviation is concentrated → 12.9x improvement

Key papers: Zou et al. (2023) "Representation Engineering", Rimsky et al. (2023) "Contrastive Activation Addition"

Attention Head Ablation Intervention

Types of Attention Heads

  • Previous-token heads: attend to the preceding token (bigram statistics)
  • Induction heads: copy patterns ("if A B... A appeared, predict B")
  • Name Mover heads: move entity names to prediction position

What we tested

Identified top 5 heads that shifted attention most during injection. Clamped their patterns to baseline values.

Result: Zero effect on GPT-2.

This is a significant null result. The injection mechanism runs through the MLP/residual pathway, not through attention redistribution. Attention shifts are correlates of deviation, not causes.
Implication: Proposed "attention-monitoring" defenses for prompt injection would miss the real mechanism on small models.
FEATURE DECOMPOSITION

Sparse Autoencoders (SAEs)

The hottest technique in MI — decomposing activations into interpretable features

The Problem: Superposition

Neural network neurons are polysemantic — each neuron responds to multiple unrelated concepts. Neuron #437 might activate for "French language" AND "ocean imagery" AND "formal tone."

Why? The superposition hypothesis (Elhage et al., 2022):

Models represent more concepts than they have neurons by encoding concepts as directions in activation space, not as individual neurons.

Like fitting 100 almost-perpendicular lines in 50-dimensional space — they overlap slightly but are approximately orthogonal.

We confirmed this!
In our neuron experiments on GPT-J, all 20 probed neurons (injection AND task) promoted the same tokens ("Ell", "Bella"). Individual neurons are NOT monosemantic.

The Solution: Sparse Autoencoders

An SAE decomposes dense, polysemantic activations into sparse, monosemantic features (each feature = one concept). 
Encoder:  features = ReLU(W_enc @ activation + b_enc)    // 768d → 25,000 features
Decoder:  reconstruction = W_dec @ features + b_dec       // 25,000 features → 768d

Loss = ||activation - reconstruction||^2 + λ × ||features||_1
                                              ^^ sparsity penalty

Key properties

  • Overcomplete: 25K features >> 768 hidden dims
  • Sparse: only 10-50 features active per input
  • Monosemantic: each feature ideally = one concept
  • Pre-trained available: no need to train your own

Variants (2024-2025)

  • TopK SAEs (OpenAI): keep only top-k features
  • JumpReLU (DeepMind): learnable threshold
  • Gated SAEs: separate gating & magnitude
  • Cross-Layer Transcoders (Anthropic): predict MLP outputs across layers

SAE Features in Practice

What we found when running SAEs on our translation prompt injection task:

Baseline ("Translate: The book...")

  • Feature #21868: "Romance language" → 3.68
  • Feature #14419: "Structured output" → 12.73
  • Feature #488: "Reference" → 9.26
  • Feature #565: "Narrative" → 0.00
  • Feature #10599: "Conversational" → 1.00

Under injection ("Ignore...")

  • Feature #21868: "Romance language" → 5.02 (UP!)
  • Feature #14419: "Structured output" → 7.44 (suppressed)
  • Feature #488: "Reference" → 1.16 (crushed)
  • Feature #565: "Narrative" → 5.68 (new!)
  • Feature #10599: "Conversational" → 8.38 (spike!)
Key insight: The model doesn't lose its "French awareness" (feature #21868 increases). It loses its task-framing features (#14419, #488) that structure the response as a translation exercise.

Few-Shot In-Context Learning

Idea: Instead of fine-tuning, provide examples in the prompt. The model learns the task "on the fly" from pattern completion. 
Translate English to French.

English: The cat is on the table.    ← example 1
French: Le chat est sur la table.

English: I love music.               ← example 2
French: J'aime la musique.

English: The book is on the shelf.   ← new input
French: ___                           ← model completes

How it works (mechanistically)

  • Induction heads: identify the pattern (English: X → French: Y)
  • Task vectors: examples create a "task direction" in the residual stream
  • Implicit fine-tuning: attention mechanism implements something like gradient descent on the examples
Our finding: More examples = stronger task vector = more injection resistance. 1-shot is 4-6x more vulnerable than 3-shot. Injection works by introducing a competing task direction in the residual stream.

The MI Tooling Ecosystem

Libraries, platforms, and infrastructure we discovered and used

NDIF: National Deep Inference Facility

What: Free research infrastructure that hosts large AI models (up to 405B parameters) for interpretability research. Run experiments on GPT-J-6B, Llama-3.1-70B, and more — from your laptop, no GPU needed.

How it works

  • Models run on NDIF's GPU cluster
  • You send "trace" requests via nnsight
  • Full access to all internal activations
  • Can read AND write to any layer
  • Free API key at login.ndif.us

Models available (hot)

  • Llama-3.1-405B, 70B, 8B
  • GPT-J-6B
  • Gemma-2-9B, Qwen2.5-7B
  • Mistral, DeepSeek, and more
import nnsight
from nnsight import LanguageModel

# Load model (runs remotely)
model = LanguageModel(
  "EleutherAI/gpt-j-6b",
  dispatch=True
)

# Read internal state
with model.trace("prompt",
                 remote=True):
  hidden = model.transformer \
    .h[24].output[0].save()
  logits = model.lm_head \
    .output.save()

# hidden is now a real tensor!

nnsight: The "Write Once, Run Anywhere" MI Library

What: General-purpose library for inspecting and intervening on any PyTorch model. Works locally (CPU/GPU) and remotely (via NDIF). ~870 GitHub stars.

Key capabilities

  • Read: save any activation from any layer
  • Write: replace activations mid-inference
  • Trace context: all operations defined declaratively
  • Remote execution: same code runs locally or on NDIF

Gotchas we discovered

  • Proxy scoping: saved values are proxies inside trace context — wrap in helper functions that return resolved tensors
  • Individual neuron indexing: complex indexing on proxies is unreliable — use full-vector injection instead
  • CPU/CUDA mismatch: use multi-invocation traces to keep tensors on the same device
# Activation patching example
def inject_vector(prompt, layer, vec):
  with model.trace(prompt,
                   remote=True,
                   scan=False,
                   validate=False):
    # Replace hidden state
    model.transformer.h[layer] \
      .output[0][:, -1, :] = vec
    # Read output
    logits = model.lm_head \
      .output[:, -1, :].save()
  return logits[0].float().detach()

TransformerLens: The MI Standard Library

What: Purpose-built MI library by Neel Nanda. Reimplements 50+ architectures with hooks at every activation. The de facto standard for local MI research. ~5K GitHub stars.

Key features

  • Hook-based access to every internal activation
  • run_with_cache() stores all activations in one call
  • Built-in logit lens, attention visualization
  • Named hooks: "blocks.8.hook_resid_post"
  • Works for GPT-2, GPT-Neo, Pythia, Llama, etc.
from transformer_lens import HookedTransformer

model = HookedTransformer.from_pretrained(
  "gpt2-small"
)

# Run and cache ALL activations
_, cache = model.run_with_cache(prompt)

# Access any layer's residual stream
resid = cache["blocks.8.hook_resid_post"]
# Shape: [batch, seq_len, 768]

# Logit lens
normed = model.ln_final(resid[0, -1])
logits = model.unembed(normed)

pip install transformer_lens

SAE Lens: Pre-trained SAEs at Your Fingertips

What: Library for training and analyzing SAEs by Joseph Bloom. Maintains a registry of pre-trained SAEs for GPT-2, Gemma, Pythia, and more. ~1.3K GitHub stars.

Pre-trained SAEs we used

  • gpt2-small-res-jb: 12 SAEs (one per layer)
  • 24,576 features each
  • Trained on OpenWebText
  • Available on HuggingFace: jbloom/GPT2-Small-SAEs

Also available for

  • Gemma 2 (Gemma Scope)
  • Pythia
  • GPT-4 (OpenAI, 16M features)
  • Claude (Anthropic, internal)
from sae_lens import SAE

# Load pre-trained SAE
sae, cfg, sparsity = SAE.from_pretrained(
  release="gpt2-small-res-jb",
  sae_id="blocks.8.hook_resid_pre",
)

# Decompose activations
features = sae.encode(hidden_state)
# features: [24576] - mostly zeros

# Get active features
active = (features > 0).nonzero()
# Typically 10-50 active features

# Reconstruct
reconstructed = sae.decode(features)

How the Techniques Connect

                OBSERVATION                      INTERVENTION
                ===========                      ============

Logit Lens       → "What does it believe?"     Head Ablation    → "Does this head matter?"
Cosine Distance  → "How different is it?"      Clamping         → "Does this layer matter?"
Entropy          → "How confident is it?"      Steering         → "Can we cancel deviation?"
Attention        → "Where does it look?"       SAE Ablation     → "Is this feature needed?"
SAE Features     → "Which concepts are active?" SAE Steering     → "Can we control precisely?"
Neuron Fingerprint→ "Which neurons fire?"       Neuron Ablation  → "Do 50 neurons suffice?"
Our progression: Coarse (whole-layer logit lens) → Medium (multi-layer steering) → Fine (SAE features) → Finest (individual neurons). Each step gave more precise understanding but also revealed deeper entanglement.
